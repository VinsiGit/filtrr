API Documentatie

Welkom bij de API-documentatie. Hieronder vindt u een overzicht van alle beschikbare endpoints en de bijbehorende queryparameters, bodyparameters en toegestane gebruikersrollen.

GET /api

Haalt de status van de API op.

Toegestane rollen:

  • Geen authenticatie vereist

POST /api/login

Verifieert de gebruiker en retourneert een JWT-token.

Body Parameters:

  • username: string (vereist) - De gebruikersnaam van de gebruiker.
  • password: string (vereist) - Het wachtwoord van de gebruiker.

Toegestane rollen:

  • Geen authenticatie vereist

GET /api/users

Haalt een lijst van alle gebruikers op.

Toegestane rollen:

  • admin

POST /api/users

Voegt een nieuwe gebruiker toe.

Body Parameters:

  • username: string (vereist) - De gebruikersnaam van de nieuwe gebruiker.
  • password: string (vereist) - Het wachtwoord van de nieuwe gebruiker.

Toegestane rollen:

  • admin

PUT /api/users

Wijzigt een bestaande gebruiker.

Body Parameters:

  • username: string (vereist) - De gebruikersnaam van de te wijzigen gebruiker.
  • password: string (optioneel) - Het nieuwe wachtwoord van de gebruiker.

Toegestane rollen:

  • admin

DELETE /api/users

Verwijdert een bestaande gebruiker.

Body Parameters:

  • username: string (vereist) - De gebruikersnaam van de te verwijderen gebruiker.

Toegestane rollen:

  • admin

GET /api/mails

Haalt een lijst van alle mails op.

Query Parameters:

  • sender: string (optioneel) - Filter op afzender van de e-mail.
  • recipient: string (optioneel) - Filter op ontvanger van de e-mail.
  • subject: string (optioneel) - Filter op onderwerp van de e-mail.
  • start_date: string (optioneel) - Filter op startdatum.
  • end_date: string (optioneel) - Filter op einddatum.

Toegestane rollen:

  • admin

GET /api/db

Haalt informatie over de database op.

Query Parameters:

  • collection: string (optioneel) - Specificeer de collectie binnen de database.
  • stats: string (optioneel) - Specificeer welke statistieken moeten worden opgevraagd (bijv. "size", "count").

Toegestane rollen:

  • admin

GET /api/stats

Haalt statistieken van de API op.

Query Parameters:

  • rating: int (optioneel) - Filter op beoordeling, standaard is "all_ratings".
  • predicted_label: string (optioneel) - Filter op voorspeld label, standaard is "all_labels".
  • actual_label: string (optioneel) - Filter op daadwerkelijk label, standaard is "all_labels".
  • source: string (optioneel) - Filter op bron, standaard is "all_sources".
  • model_version: float (optioneel) - Filter op modelversie, standaard is "all_versions".
  • start_date: string (optioneel) - Filter op startdatum.
  • end_date: string (optioneel) - Filter op einddatum.

Toegestane rollen:

  • admin
  • user

GET /api/cnfmtrx

Haalt de verwarringsmatrix op.

Toegestane rollen:

  • admin
  • user

GET /api/stats/labels

Haalt de labels van statistieken op.

Toegestane rollen:

  • admin
  • user

POST /api

Algemene POST endpoint voor de API.

Body Parameters:

  • Afhankelijk van de specifieke functionaliteit.

Toegestane rollen:

  • admin
  • user

PUT /api

Algemene PUT endpoint voor de API.

Body Parameters:

  • Afhankelijk van de specifieke functionaliteit.

Toegestane rollen:

  • admin

GET /api/settings

Haalt de instellingen van de API op.

Toegestane rollen:

  • admin

GET /api/retrain

Start het retrain proces.

Toegestane rollen:

  • admin

GET /api/tokencheck

Controleert de geldigheid van het token.

Toegestane rollen:

  • admin
  • user
